# Экономный режим

Используй, когда пользователь хочет экономить токены или сообщения.

Правила:
1. Сначала найди релевантные файлы через поиск.
2. Не открывай большие файлы полностью, если достаточно конкретного фрагмента.
3. Не переписывай файлы целиком без необходимости.
4. Делай изменения маленькими порциями.
5. Не печатай полный код в ответе.
6. Не запускай тяжелые проверки после каждого мелкого изменения; группируй проверки.
7. Сначала запускай быстрые проверки, потом дорогие:
   - typecheck
   - unit
   - build
   - e2e
   - moderation check
8. Если задача большая, раздели ее на этапы:
   - диагностика;
   - план;
   - реализация;
   - тесты;
   - финальная проверка.
9. В ответе показывай только:
   - что изменено;
   - какие файлы затронуты;
   - какие команды прошли;
   - какие проблемы остались.
