# AGENTS.md — правила проекта Яндекс Игры

## Команды проекта
- Установка: npm install
- Запуск разработки: npm run dev
- Сборка: npm run build
- Unit-тесты: npm run test
- E2E/UI-тесты: npm run test:e2e
- Проверка модерации: npm run check:yandex
- Сборка архива: npm run build:zip

## Архитектура
- Не добавляй backend, сервер, базы данных и внешние API.
- Весь прогресс хранится локально и через SDK, если он доступен.
- Не разносить одну механику по десяткам файлов без необходимости.

## Интерфейс
- Все тексты интерфейса только на русском.
- Минимальный размер основного текста: 18 px.
- Минимальный размер кнопки: 44×44 px.
- На маленьких окнах интерфейс должен перестраиваться, а не обрезаться.
- Нельзя допускать наложение текста, кнопок, модальных окон и игрового поля.
- Не использовать браузерные alert, prompt, confirm.

## Модерация Яндекс Игр
Перед финальным ответом обязательно проверить:
- Нет системной прокрутки.
- Нет выделения текста или открытия контекстного меню на игровом поле.
- Нет обрезки элементов при изменении размера окна.
- Нет наложения внутренних элементов и текстов.
- Нет технических ошибок на старте.
- В корне сборочного архива есть index.html.
- В именах файлов и папок нет пробелов и русских символов.
- Размер распакованной сборки меньше 100 МБ.
- SDK подключается корректно.
- LoadingAPI.ready вызывается только после готовности игры.
- Реклама не ломает игровой цикл.
- Игра выглядит завершенной.

## Тестирование
- После изменения логики запускать unit-тесты.
- После изменения интерфейса запускать e2e/UI-тесты.
- После изменения сборки запускать build.
- После изменения SDK или рекламы запускать check:yandex.
- Если тест упал, сначала исправить, потом снова запустить.

## Ответ пользователю
- Отвечай кратко.
- Не вставляй полный код измененных файлов.
- Укажи список измененных файлов.
- Укажи команды, которые были запущены.
- Укажи результат проверок.
- Если что-то не удалось проверить, честно напиши.
